# QueryEngine 功能、意图与实现分析

`QueryEngine` 是一个深度研究代理，它接收用户查询，通过多轮搜索、反思和总结来生成综合报告。

## 核心功能:

1.  **接收用户查询:** 接收一个自然语言查询，例如“分析最近的AI芯片市场趋势”。
2.  **生成报告结构:** 首先，它会把大的查询分解成多个子主题，形成报告的段落结构。
3.  **分段研究:**
    *   **初始搜索:** 对每个子主题，它会生成搜索关键词，并选择最合适的搜索工具（如基础新闻搜索、按日期搜索等）进行网络搜索。
    *   **初始总结:** 根据搜索结果，生成该子主题的初步摘要。
    *   **反思与迭代:** 对初步摘要进行“反思”，评估其内容的深度和广度，然后进行新一轮的、更有针对性的搜索来补充信息。这个过程会重复多次（`MAX_REFLECTIONS`）。
    *   **迭代总结:** 每次反思搜索后，都会更新和完善子主题的摘要。
4.  **生成最终报告:** 所有子主题都研究完毕后，会将所有段落的最终摘要整合成一份结构完整、内容丰富的Markdown格式报告。
5.  **保存结果:** 将最终报告和中间研究状态（可选）保存到文件系统。

## 意图:

`QueryEngine` 的设计意图是模拟一个人类研究员的工作流程。它不是简单地返回一堆搜索链接，而是通过一个结构化的“思考”过程——分解问题、搜索、总结、反思、再搜索——来产出一份深度和广度都经过优化的研究报告。这使其能够处理更复杂、更开放式的查询。

## 实现:

*   **`DeepSearchAgent` 类:** 这是核心控制器，负责协调整个研究流程。
*   **状态机 (`State`):** 用一个 `State` 对象来跟踪整个研究过程，包括报告结构、每个段落的搜索历史、总结内容等。这使得整个流程清晰、可追溯，并且支持中断和恢复。
*   **节点 (`Nodes`):** 将研究过程中的每个逻辑步骤（如生成报告结构、初次搜索、反思、总结）都封装成独立的“节点” (`ReportStructureNode`, `FirstSearchNode`, `ReflectionNode` 等)。每个节点接收当前状态，执行自己的任务，然后输出更新后的状态。这种模块化设计使得代码更清晰，易于扩展和维护。
*   **工具 (`Tools`):** 集成了 `TavilyNewsAgency` 作为其强大的搜索引擎，提供了多种搜索模式（基础搜索、深度搜索、按时间搜索等），`DeepSearchAgent` 会根据需要智能选择最合适的工具。
*   **大语言模型 (`LLMClient`):** 在整个流程中深度使用大语言模型（通过 `LLMClient`）来完成需要“理解”和“生成”的任务，例如：
    *   将用户查询分解为报告结构。
    *   根据子主题生成搜索关键词。
    *   根据搜索结果生成摘要。
    *   对现有摘要进行反思，提出下一步研究方向。
    *   将所有段落整合成最终报告。

总的来说，`QueryEngine` 是一个精心设计的自动化研究代理，它结合了状态管理、模块化节点、外部工具和强大的语言模型，以一种系统化、可迭代的方式来完成深度研究任务。

# `QueryEngine` 调度线索与模拟输入输出

假设我们向 `QueryEngine` 发出以下研究请求：

**用户查询 (Query):** `"分析最近的AI芯片市场趋势"`

整个流程将围绕一个核心的 `State` 对象进行，它会像一个动态的研究档案，在每一步中被不断填充和更新。

---

### 初始状态

在 `agent.research(query)` 被调用时，`State` 对象被初始化，但大部分是空的。

```json
// 初始 State 对象 (简化)
{
  "query": "分析最近的AI芯片市场趋势",
  "report_title": "",
  "paragraphs": [],
  "final_report": "",
  "status": "IN_PROGRESS"
}
```

---

### 步骤 1: 生成报告结构 (`_generate_report_structure`)

**目标:** 将模糊的用户查询分解成一个具体的、有逻辑的报告大纲。

1.  **调用节点:** `ReportStructureNode`
2.  **输入:**
    *   `query`: `"分析最近的AI芯片市场趋势"`
3.  **LLM 交互 (模拟):**
    *   **发送给 LLM 的 Prompt (意译):**
        > "你是一个专业的研究助理。请为以下研究主题创建一个详细的报告大纲：'分析最近的AI芯片市场趋势'。请以JSON格式返回报告标题和段落结构，每个段落应包含一个标题(title)和一个简短的内容描述(content)，说明该段落将要探讨的重点。"
4.  **LLM 返回的输出 (模拟):**

```json
    {
      "report_title": "近期AI芯片市场趋势综合分析报告",
      "paragraphs": [
        {
          "title": "主要参与者及其最新动态",
          "content": "分析NVIDIA, Google, Intel, AMD等主要厂商在AI芯片领域的最新产品发布、市场份额和战略举措。"
        },
        {
          "title": "新兴技术与架构创新",
          "content": "探讨Chiplet、存内计算、神经形态计算等新兴技术对AI芯片性能和效率的影响。"
        },
        {
          "title": "市场面临的挑战与机遇",
          "content": "分析供应链安全、功耗墙、以及软件生态带来的挑战，同时探讨边缘计算、AI PC等领域带来的新机遇。"
        },
        {
          "title": "未来发展方向预测",
          "content": "结合当前趋势，预测未来3-5年AI芯片技术和市场的可能发展方向。"
        }
      ]
    }
```

5.  **状态更新:** `State` 对象被LLM的返回结果填充。

```json
    // 步骤1结束后的 State 对象
    {
      "query": "分析最近的AI芯片市场趋势",
      "report_title": "近期AI芯片市场趋势综合分析报告",
      "paragraphs": [
        { "title": "主要参与者及其最新动态", "content": "...", "research": { "search_history": [], "initial_summary": "", "latest_summary": "" } },
        { "title": "新兴技术与架构创新", "content": "...", "research": { "search_history": [], "initial_summary": "", "latest_summary": "" } },
        // ... 其他段落
      ],
      "final_report": "",
      "status": "IN_PROGRESS"
    }
```

---

### 步骤 2: 处理每个段落 (`_process_paragraphs`)

这是一个大循环，我们将以**第一个段落“主要参与者及其最新动态”**为例，详细拆解其内部流程。

#### 步骤 2.1: 初始搜索与总结 (`_initial_search_and_summary`)

**目标:** 对段落主题进行第一次网络搜索，并形成初步摘要。

1.  **调用节点 (生成搜索词):** `FirstSearchNode`
2.  **输入:**
    *   `title`: `"主要参与者及其最新动态"`
    *   `content`: `"分析NVIDIA, Google, Intel, AMD等主要厂商..."`
3.  **LLM 交互 (模拟):**
    *   **Prompt:**
        > "为研究段落'主要参与者及其最新动态'，请生成一个高效的搜索查询(search_query)和最适合的搜索工具(search_tool)。说明你的推理(reasoning)。"
    *   **LLM 输出:**

```json
        {
          "search_query": "NVIDIA H200, AMD MI300X, Intel Gaudi 3, Google TPU v5p latest news and market share",
          "search_tool": "basic_search_news",
          "reasoning": "该查询具体指明了主要厂商的最新芯片型号，能有效获取相关新闻和市场分析。基础新闻搜索工具足以满足首次信息获取的需求。"
        }
```

4.  **执行搜索:**
    *   `execute_search_tool("basic_search_news", "NVIDIA H200...")` 被调用。
    *   `TavilyNewsAgency` 工具执行搜索，返回一个搜索结果列表。
    *   **搜索结果 (模拟):**

```
        [
          { "title": "NVIDIA Blackwell Platform Arrives...", "url": "...", "content": "NVIDIA today announced the Blackwell platform... delivering 2.5x the performance of the H100..." },
          { "title": "AMD Instinct MI300X: A Strong Competitor...", "url": "...", "content": "The MI300X accelerator is showing strong performance in benchmarks against NVIDIA's H100..." },
          ...
        ]
```

5.  **调用节点 (生成总结):** `FirstSummaryNode`

6.  **输入:**
    *   段落标题、内容描述
    *   搜索查询: `"NVIDIA H200..."`
    *   搜索结果: `[ { "title": "...", "content": "..." }, ... ]`
7.  **LLM 交互 (模拟):**
    *   **Prompt:**
        > "根据以下搜索结果，为段落'主要参与者及其最新动态'撰写一段总结。"
    *   **LLM 输出 (一段文本):**
        > "AI芯片市场的主要参与者NVIDIA和AMD近期均有重要发布。NVIDIA推出了性能强大的Blackwell平台，其性能据称是H100的2.5倍。与此同时，AMD的MI300X加速器在基准测试中也展现出对NVIDIA H100的强大竞争力..."
8.  **状态更新:** 第一个段落的 `research` 部分被更新。

```json
    // 第一个段落的状态
    "research": {
      "search_history": [
        { "query": "NVIDIA H200...", "results": [...] }
      ],
      "initial_summary": "AI芯片市场的主要参与者...",
      "latest_summary": "AI芯片市场的主要参与者..." // 此刻与 initial_summary 相同
    }
```

#### 步骤 2.2: 反思循环 (`_reflection_loop`)

**目标:** 批判性地审视现有总结，发现不足，并通过新一轮的搜索来弥补信息。这个循环会重复几次。

**第一次反思:**

1.  **调用节点:** `ReflectionNode`
2.  **输入:**
    *   段落标题
    *   `latest_summary`: `"AI芯片市场的主要参与者..."`
3.  **LLM 交互 (模拟):**
    *   **Prompt:**
        > "审阅以下关于'主要参与者'的总结。它缺少了哪些关键信息？哪些问题还没有回答？基于你的批判性思考，生成一个新的、更具探索性的搜索查询来弥补这些不足。"
    *   **LLM 输出:**

```json
        {
          "search_query": "emerging AI chip startups and custom silicon from cloud providers like AWS Graviton",
          "reasoning": "目前的总结只关注了NVIDIA和AMD，忽略了市场上重要的初创公司（如Cerebras, SambaNova）以及云服务商（如AWS, Google）自研芯片的趋势，这些也是'主要参与者'的一部分。"
        }
```

4.  **执行搜索 & 生成反思总结:**
    *   使用新的查询 `"emerging AI chip startups..."` 进行搜索。
    *   **调用节点:** `ReflectionSummaryNode`
    *   **输入:**
        *   `latest_summary` (旧的总结)
        *   新的搜索结果 (关于初创公司和云厂商自研芯片)
    *   **LLM 交互 (Prompt):**
        > "这是你之前的总结：'[旧总结]'。现在，请将以下关于初创公司和云厂商自研芯片的新信息整合进去，生成一个更全面、更完善的新版本。"
    *   **LLM 输出 (一段更新后的文本):**
        > "AI芯片市场由NVIDIA和AMD等传统巨头主导，NVIDIA的Blackwell平台和AMD的MI300X展现了激烈的性能竞争。**此外，市场格局正变得更加多元化。一方面，像Cerebras和SambaNova这样的初创公司通过创新的架构获得了关注。另一方面，大型云服务商如AWS和Google正通过自研芯片（如Graviton和TPU）来优化其云服务的成本和效率，成为不可忽视的力量。**"
5.  **状态更新:** 第一个段落的 `latest_summary` 被新生成的、更完善的文本覆盖。`search_history` 也被追加了新的搜索记录。这个反思循环会继续执行，直到达到 `MAX_REFLECTIONS` 次数，使得总结越来越完善。

---

### 步骤 3: 生成最终报告 (`_generate_final_report`)

**目标:** 将所有段落的最终研究成果 (`latest_summary`) 组合成一份完整的报告。

1.  **调用节点:** `ReportFormattingNode`
2.  **输入:** 一个包含所有段落最终总结的列表。

```
    [
      { "title": "主要参与者及其最新动态", "paragraph_latest_state": "AI芯片市场由NVIDIA和AMD..." },
      { "title": "新兴技术与架构创新", "paragraph_latest_state": "Chiplet技术正成为主流..." },
      ...
    ]
```

3.  **LLM 交互 (模拟):**
    *   **Prompt:**
        > "你是一位专业的报告撰写专家。请将以下各段落的内容整合成一份流畅、连贯、格式精美的Markdown报告。报告标题为：'近期AI芯片市场趋势综合分析报告'。请确保段落间有自然的过渡。"
4.  **LLM 输出 (最终报告的Markdown文本):**

```
    # 近期AI芯片市场趋势综合分析报告

    ## 主要参与者及其最新动态

    AI芯片市场由NVIDIA和AMD等传统巨头主导，NVIDIA的Blackwell平台和AMD的MI300X展现了激烈的性能竞争。此外，市场格局正变得更加多元化...

    ## 新兴技术与架构创新

    在技术层面，Chiplet技术正成为主流，它允许厂商像搭积木一样组合不同的功能模块...

    ...
```

5.  **状态更新:** `State` 对象的 `final_report` 字段被填充。

---

### 步骤 4: 保存报告 (`_save_report`)

**目标:** 将最终报告写入文件。

1.  **输入:** `report_content` (上一步生成的Markdown文本)。
2.  **操作:**
    *   生成文件名，如 `deep_search_report_分析最近的AI芯片市场趋势_20231027_103000.md`。
    *   将Markdown文本写入到 `OUTPUT_DIR` 目录下的这个文件中。
    *   (可选) 将最终的 `State` 对象序列化为JSON，保存为 `state_... .json` 文件，用于调试和复盘。

至此，一次完整的深度研究流程就结束了。
