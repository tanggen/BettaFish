# `InsightEngine` 调度线索与模拟输入输出

`InsightEngine` 是一个针对**私有数据库**进行深度挖掘和洞察的代理。它通过**查询优化**、**结果聚类**和**情感分析**等特色功能，从海量内部数据中提取有价值的信息。

以下是其处理用户查询 `"最近一周关于我们新产品'星尘'的用户反馈怎么样？"` 的模拟流程。

---

### 初始状态

`agent.research(query)` 被调用，创建一个空的 `State` 对象来记录整个分析过程。

```json
// 初始 State 对象 (简化)
{
  "query": "最近一周关于我们新产品'星尘'的用户反馈怎么样？",
  "report_title": "",
  "paragraphs": [],
  "final_report": "",
  "status": "IN_PROGRESS"
}
```

---

### 步骤 1: 生成报告结构 (`_generate_report_structure`)

**目标:** 将一个开放式问题转化为结构化的分析报告大纲。

1.  **调用节点:** `ReportStructureNode`
2.  **输入:**
    *   `query`: `"最近一周关于我们新产品'星尘'的用户反馈怎么样？"`
3.  **LLM 交互 (模拟):**
    *   **Prompt (意译):**
        > "你是一个专业的市场分析师。请为以下分析任务创建一个详细的报告大纲：'分析最近一周关于新产品'星尘'的用户反馈'。请以JSON格式返回报告标题和段落结构，每个段落应包含标题(title)和内容描述(content)。"
4.  **LLM 返回 (模拟):**

```json
    {
      "report_title": "新产品'星尘'近一周用户反馈洞察报告",
      "paragraphs": [
        {
          "title": "整体情感倾向分析",
          "content": "综合评估用户反馈的整体情感分布，包括正面、负面和中性评价的比例，并给出总体结论。"
        },
        {
          "title": "主要赞扬点",
          "content": "挖掘用户最常提到的正面反馈，识别产品的核心吸引力在哪些功能或特性上。"
        },
        {
          "title": "主要批评与改进建议",
          "content": "识别用户集中抱怨的问题点（如Bug、性能、设计缺陷），并整理用户提出的具体改进建议。"
        },
        {
          "title": "高价值用户个案摘录",
          "content": "挑选几条具有代表性的、详细的用户评论（包括正面和负面），作为具体例证。"
        }
      ]
    }
```

5.  **状态更新:** `State` 对象的 `paragraphs` 被填充，为后续的深入分析设定了框架。

---

### 步骤 2: 处理每个段落 (`_process_paragraphs`)

我们将以**第三个段落“主要批评与改进建议”**为例，详细拆解其内部复杂的处理流程。

#### 步骤 2.1: 初始搜索与总结 (`_initial_search_and_summary`)

**目标:** 从数据库中提取与“批评和建议”相关的数据，并形成初步摘要。

1.  **调用节点 (生成搜索策略):** `FirstSearchNode`
2.  **输入:**
    *   `title`: `"主要批评与改进建议"`
    *   `content`: `"识别用户集中抱怨的问题点..."`
3.  **LLM 交互 (模拟):**
    *   **Prompt:**
        > "为研究'星尘'产品的'主要批评与改进建议'，请生成一个搜索查询(search_query)和最适合的数据库查询工具(search_tool)，并提供时间范围。"
    *   **LLM 输出:**

```json
        {
          "search_query": "星尘",
          "search_tool": "search_topic_by_date",
          "start_date": "2023-10-20", 
          "end_date": "2023-10-27",
          "reasoning": "需要在一个特定的时间范围内搜索关于'星尘'的所有反馈，以便后续进行筛选和分析。search_topic_by_date是最佳工具。"
        }
```

4.  **执行搜索 (`execute_search_tool`):** 这是 `InsightEngine` 最核心和复杂的步骤。
    *   **a. 关键词优化 (`keyword_optimizer`):**
        *   **输入:** 原始查询 `"星尘"`。
        *   **LLM 交互:**
            > "为了找到关于'星尘'的批评和建议，请将关键词'星尘'扩展为一组更具体的、可能与问题相关的搜索词。"
        *   **LLM 返回:**
            `{ "optimized_keywords": ["星尘 耗电", "星尘 卡顿", "星尘 闪退", "星尘 建议", "星尘 bug", "星尘 不好用"], "reasoning": "..." }`
    *   **b. 数据库查询:** Agent会遍历优化后的关键词，使用 `search_topic_by_date` 工具在数据库中执行多次查询。
    *   **c. 结果汇总与去重:** 假设从数据库共返回了 **800** 条相关结果，去重后得到 **650** 条独立反馈。
    *   **d. 聚类采样 (`_cluster_and_sample_results`):**
        *   **输入:** 650条用户反馈文本。
        *   **处理:**
            1.  使用 `paraphrase-multilingual-MiniLM-L12-v2` 模型将650条文本编码为向量。
            2.  运行K-Means算法，将这些向量聚成（例如）10个簇。
            3.  从每个簇中，根据热度或相关性选出5条最具代表性的反馈。
        *   **输出:** 一个大幅精简但多样性得以保留的列表，包含 **50** 条代表性结果。
    *   **e. 情感分析 (`_perform_sentiment_analysis`):**
        *   **输入:** 50条采样后的结果。
        *   **处理:** 调用 `multilingual_sentiment_analyzer` 对每条文本进行情感打分。
        *   **输出:** 一个包含情感统计的分析对象，并附加到最终的搜索响应中。例如：`{ "summary": { "negative": 42, "neutral": 5, "positive": 3 }, ... }`

5.  **调用节点 (生成总结):** `FirstSummaryNode`
6.  **输入:**
    *   段落标题、内容
    *   经过**优化、搜索、聚类、采样、情感分析**后得到的50条高质量数据。
7.  **LLM 交互 (模拟):**
    *   **Prompt:**
        > "根据以下用户反馈数据（已进行聚类和情感分析），总结关于'星尘'产品的主要批评和改进建议。"
    *   **LLM 输出 (初步总结):**
        > "用户反馈中，主要的批评集中在电池续航上，许多用户反映在正常使用下耗电过快。其次，部分用户报告了在切换应用时偶尔出现的卡顿现象。在改进建议方面，用户普遍希望能够增加夜间模式的自定义选项，并希望优化通知管理功能。"

8.  **状态更新:** 第三个段落的 `research.initial_summary` 和 `research.latest_summary` 被填充。

#### 步骤 2.2: 反思循环 (`_reflection_loop`)

**目标:** 审视初步总结，发现更深层次的问题，并进行追问式搜索。

1.  **调用节点:** `ReflectionNode`
2.  **输入:** 上一步生成的 `latest_summary`。
3.  **LLM 交互 (模拟):**
    *   **Prompt:**
        > "审阅此总结：'[初步总结]'。这个总结还有哪些模糊不清的地方？为了让分析更深入，下一步应该具体搜索什么？"
    *   **LLM 输出:**

```json
        {
          "search_query": "星尘 卡顿 手机型号",
          "search_tool": "search_topic_on_platform",
          "platform": "App Store",
          "reasoning": "总结中提到了'卡顿现象'，但这是一个非常笼统的描述。为了定位问题，我们需要知道这种现象主要发生在哪些手机型号上，以及在哪个平台的反馈中被提及最多。因此，我将使用平台定向搜索工具进行更精确的查询。"
        }
```

4.  **执行新一轮搜索 & 总结:**
    *   Agent 再次执行 `execute_search_tool` 流程（关键词优化 -> 搜索 -> 聚类 -> ...）。
    *   **调用节点:** `ReflectionSummaryNode`
    *   **LLM 交互:**
        > "这是你之前的总结：'[初步总结]'。现在，根据这份关于'卡顿'问题在不同手机型号上表现的新数据，更新你的总结，使其更加具体。"
    *   **LLM 输出 (更新后的总结):**
        > "...其次，部分用户报告了在切换应用时偶尔出现的卡顿现象，**根据App Store的评论数据分析，此问题更多地集中在三年前发布的旧款机型上**。在改进建议方面..."

5.  **状态更新:** `latest_summary` 被这个更详细、更具洞察力的版本覆盖。

---

### 步骤 3 & 4: 生成并保存最终报告

1.  **生成报告 (`_generate_final_report`):**
    *   **调用节点:** `ReportFormattingNode`
    *   **输入:** 所有段落的最终 `latest_summary`。
    *   **LLM 输出:** 一份结构完整、用词专业的Markdown报告。
2.  **保存报告 (`_save_report`):**
    *   将生成的Markdown文本保存到 `insight_engine_streamlit_reports` 目录下，文件名如 `deep_search_report_最近一周关于我们新产品_20231027_110000.md`。
    *   同时，将包含所有中间步骤的最终 `State` 对象保存为JSON文件，以备复盘。

至此，`InsightEngine` 完成了一次从原始数据到深度洞察的完整分析流程。
