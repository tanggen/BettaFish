# `ForumEngine` 调度线索与模拟输入输出

`ForumEngine` 是整个系统的“中枢神经”，一个实现多Agent协作的“会议室”。它的工作流程不是线性的，而是事件驱动的，以下是其典型的调度线索。

#### 场景设定

*   `QueryEngine`, `InsightEngine`, `MediaEngine` 都在运行，但处于空闲状态。
*   `ForumEngine` (`monitor.py`) 作为一个后台线程已经启动，正在“打瞌睡”，持续检查三个引擎的日志文件 (`query.log`, `insight.log`, `media.log`) 是否有变化。
*   `forum.log` 文件是空的，或者只包含一行“监控开始”的系统消息。

---

### 步骤 1: “会议开始” - 首次发言的捕获

**目标:** 识别出新一轮研究的开始，并准备记录。

1.  **外部事件:** 用户通过主应用 `app.py` 发起了一个研究请求，例如：“分析AI芯片市场”。这个请求被分发给了 `QueryEngine` 和 `InsightEngine`。

2.  **日志文件变化:**
    *   `QueryEngine` 开始工作，执行它的 `research` 流程。在完成“初始搜索与总结”后，它向自己的日志文件 `query.log` 写入了第一条总结信息。
    *   **`query.log` 中出现新内容 (模拟):**
        ```
        2023-10-27 10:30:00.123 | INFO | QueryEngine.nodes.summary_node:run:265 - 正在生成首次段落总结...
        2023-10-27 10:30:01.456 | INFO | QueryEngine.nodes.summary_node:process_output:131 - 清理后的输出: { "paragraph_latest_state": "AI芯片市场目前由NVIDIA和AMD主导，两者竞争激烈..." }
        ```

3.  **`ForumEngine` 的反应 (`monitor_logs` 循环):**
    *   `monitor.py` 的监控线程检测到 `query.log` 文件变大了。
    *   它读取新增的几行日志。
    *   `is_target_log_line` 函数被调用，当读到包含 `summary_node` 和 `正在生成首次段落总结` 的行时，返回 `True`。
    *   **关键判断:** 因为这是 `FirstSummaryNode` 的输出，`ForumEngine` 判定**一场新的研讨会开始了**。
    *   `self.is_searching` 状态被设置为 `True`。
    *   `clear_forum_log()` 被调用，`forum.log` 被清空并写入一行系统消息。
    *   **`forum.log` 的内容:**
        ```
        [10:30:01] [SYSTEM] === ForumEngine 监控开始 - 2023-10-27 10:30:01 ===
        ```

---

### 步骤 2: “会议记录” - 捕获并转录发言

**目标:** 解析出各个Agent的总结内容，并统一记录到 `forum.log`。

1.  **`ForumEngine` 继续处理日志:**
    *   `process_lines_for_json` 函数被调用来处理刚才读取到的新日志行。
    *   它识别出包含 `"清理后的输出: {"` 的行，并智能地捕获了完整的JSON对象（无论是单行还是多行）。
    *   `extract_json_content` 函数从JSON中提取出核心的 `"paragraph_latest_state"` 的值。
    *   `write_to_forum_log` 被调用。

2.  **`forum.log` 更新:**
    *   `ForumEngine` 将提取出的内容，附上来源 `[QUERY]`，写入 `forum.log`。
    *   **`forum.log` 的新内容:**
        ```
        [10:30:01] [SYSTEM] === ForumEngine 监控开始 - 2023-10-27 10:30:01 ===
        [10:30:01] [QUERY] AI芯片市场目前由NVIDIA和AMD主导，两者竞争激烈...
        ```
    *   同时，这条发言被存入 `self.agent_speeches_buffer` 缓冲区。

---

### 步骤 3: “交叉讨论” - 其他Agent的发言

**目标:** 记录来自不同Agent的观点，形成讨论流。

1.  **外部事件:** 与此同时，`InsightEngine` 也完成了它自己的第一轮分析，并向 `insight.log` 写入了它的总结。
    *   **`insight.log` 中出现新内容 (模拟):**
        ```
        2023-10-27 10:30:05.789 | INFO | InsightEngine.nodes.summary_node:process_output:131 - 清理后的输出: { "paragraph_latest_state": "根据我们的内部数据显示，'星尘'产品的用户反馈中，关于耗电的负面提及率在过去一周上升了30%..." }
        ```

2.  **`ForumEngine` 的反应:**
    *   在其下一个监控周期（通常是1秒后），它检测到 `insight.log` 的变化。
    *   它重复**步骤 2** 的过程：读取 -> 解析 -> 提取 -> 写入。

3.  **`forum.log` 再次更新:**
    *   **`forum.log` 的内容:**
        ```
        [10:30:01] [SYSTEM] === ForumEngine 监控开始 - 2023-10-27 10:30:01 ===
        [10:30:01] [QUERY] AI芯片市场目前由NVIDIA和AMD主导，两者竞争激烈...
        [10:30:05] [INSIGHT] 根据我们的内部数据显示，'星尘'产品的用户反馈中，关于耗电的负面提及率在过去一周上升了30%...
        ```
    *   `InsightEngine` 的发言也被加入到 `agent_speeches_buffer`。

---

### 步骤 4: “主持人介入” - LLM的引导

**目标:** 在讨论进行到一定程度时，引入LLM主持人来总结并引导下一步方向。

1.  **内部触发条件:**
    *   `ForumEngine` 持续重复**步骤2和3**，不断记录来自各个Engine的“反思总结” (`ReflectionSummaryNode`)。
    *   假设 `agent_speeches_buffer` 中已经积累了5条发言。

2.  **`ForumEngine` 的行动:**
    *   `_trigger_host_speech()` 函数被调用。
    *   它将缓冲区中的5条发言（完整的日志行）作为上下文，传递给 `llm_host.py` 中的 `generate_host_speech` 函数。

3.  **`llm_host` 的内部工作:**
    *   **输入:** 最近的5条讨论记录。
    *   **LLM 交互 (模拟):**
        *   **Prompt (意译):**
            > "你是一个会议主持人。以下是几位专家的发言：\n1. [QUERY] AI市场由NVIDIA主导...\n2. [INSIGHT] 内部数据显示'星尘'耗电问题突出...\n3. [QUERY] NVIDIA的新架构能效比提升了50%...\n4. [MEDIA] 社交媒体上关于'星尘'耗电的讨论热度很高...\n5. [INSIGHT] 耗电问题主要出现在旧款机型上...\n\n请根据以上讨论，进行总结，并提出一个能启发大家进行更深入探讨的问题或方向。"
        *   **LLM 返回 (模拟):**
            > "总结得很好。我们已经从市场（Query）、产品（Insight）和舆论（Media）三个角度确认了'星尘'的耗电问题，特别是针对旧款机型。一个关键问题是：**这究竟是软件优化不足，还是硬件限制导致的必然结果？** InsightEngine 能否深挖一下具体的技术反馈？QueryEngine 能否查找一下竞品在旧款机型上的能耗表现作为对比？"

4.  **`forum.log` 最终更新:**
    *   `generate_host_speech` 返回的这段文本被 `ForumEngine` 捕获。
    *   `write_to_forum_log` 被调用，但这次的来源是 `[HOST]`。
    *   **`forum.log` 的内容:**
        ```
        ... (之前的发言) ...
        [10:31:00] [HOST] 总结得很好。我们已经从市场（Query）、产品（Insight）和舆论（Media）三个角度确认了'星尘'的耗电问题...
        ```

---

### 步骤 5: “会议结束”

**目标:** 在所有活动停止后，标记本次研讨会的结束。

1.  **外部事件:** `QueryEngine` 和 `InsightEngine` 等都完成了它们的 `research` 任务，不再向日志文件写入新内容。
2.  **`ForumEngine` 的反应:**
    *   `monitor_logs` 循环了多次，但发现所有被监控的日志文件大小都不再变化。
    *   `self.search_inactive_count` 计数器不断增加。
    *   当计数器达到一个阈值（例如7200次，约2小时）时，`ForumEngine` 判定会议已自然结束。
    *   `self.is_searching` 被重置为 `False`。
    *   `write_to_forum_log` 被最后一次调用。
    *   **`forum.log` 的最后一行:**
        ```
        [12:30:00] [SYSTEM] === ForumEngine 论坛结束 - 2023-10-27 12:30:00 ===
        ```

至此，`ForumEngine` 完成了一次完整的“会议”记录与主持工作，`forum.log` 文件成为这次多Agent协作过程的完整档案。
