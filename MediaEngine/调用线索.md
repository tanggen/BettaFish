我们来将 `MediaEngine` 的执行过程进行拆解，并模拟每一步的输入与输出。这会像一个“代码沙盘推演”，让你清晰地看到数据是如何在系统中流动的。

我们将以一个具体的研究任务为例：**“分析一下2024年第一季度新能源汽车市场的舆论趋势”**。

---

### **调用入口**

一切始于外部调用 `DeepSearchAgent` 实例的 `research` 方法。

```python
# 外部调用
agent = DeepSearchAgent()
query = "分析一下2024年第一季度新能源汽车市场的舆论趋势"
final_report = agent.research(query=query)
```

---

### **第一步: `_generate_report_structure` - 生成报告结构**

1.  **调用线索**: `research()` -> `_generate_report_structure()` -> `ReportStructureNode.mutate_state()`
2.  **核心节点**: `ReportStructureNode`
3.  **输入**:
    *   用户查询: `"分析一下2024年第一季度新能源汽车市场的舆论趋势"`
4.  **内部逻辑**:
    *   `ReportStructureNode` 会调用 LLM，并给出一个类似这样的指令（Prompt）:
        > "根据以下研究查询，请为一份深度研究报告生成一个合适的标题和章节结构。请以JSON格式返回，包含'report_title'和'paragraphs'列表，每个段落应有'title'和'content'（内容为你对该章节的简要描述）。查询：'分析一下2024年第一季度新能源汽车市场的舆论趋势'"
5.  **模拟输出 (LLM返回的JSON)**:
    ```json
    {

      "paragraphs": [
        {

          "content": "分析2024年Q1期间，新能源汽车在各大社交媒体和新闻平台上的整体讨论量、热搜情况及月度变化趋势。"
        },
        {
          "title": "主要品牌声量与口碑对比",
          "content": "对比分析比亚迪、特斯拉、蔚来、小鹏、理想等主流品牌的网络声量（提及量）和正负面口碑评价。"
        },
        {
          "title": "消费者核心关注点分析",
          "content": "挖掘消费者在讨论新能源汽车时最关心的话题，例如：价格战、电池续航、充电便利性、智能化功能、安全性等。"

        {
          "title": "关键技术与行业动态讨论",

    }
    ```
6.  **状态更新**: `self.state` 对象被初始化，`state.paragraphs` 列表中现在包含了5个段落对象，每个对象都有了标题和内容描述。

---

### **第二步: `_process_paragraphs` - 循环处理每个段落**

系统会遍历上一步生成的5个段落，我们以处理**第一个段落**为例：`"市场总体热度与关注度变化"`。

#### **2.1 `_initial_search_and_summary` - 首次搜索与总结**

1.  **调用线索**: `_process_paragraphs()` -> `_initial_search_and_summary()` -> `first_search_node.run()` -> `execute_search_tool()` -> `first_summary_node.mutate_state()`
2.  **核心节点**: `FirstSearchNode`, `FirstSummaryNode`
3.  **内部逻辑与模拟输出**:
    *   **a. 生成搜索查询 (`first_search_node`)**:
        *   **输入**: 段落标题 `"市场总体热度与关注度变化"`。
        *   **LLM指令**: > "为了撰写关于'市场总体热度与关注度变化'的章节，我应该使用什么搜索工具和关键词？请以JSON格式返回'search_query', 'search_tool', 'reasoning'。"
        *   **模拟输出**:
            ```json
            {
              "search_query": "2024年Q1 新能源汽车 市场热度 舆情分析",
              "search_tool": "comprehensive_search",
              "reasoning": "需要获取全面的新闻、社交媒体的综合信息来了解总体热度，因此使用综合搜索。"
            }
            ```
    *   **b. 执行搜索 (`execute_search_tool`)**:
        *   **输入**: `tool_name="comprehensive_search"`, `query="2024年Q1 新能源汽车 市场热度 舆情分析"`。
        *   **内部逻辑**: 调用 `BochaMultimodalSearch` 的 API。
        *   **模拟输出 (搜索结果)**:
            ```json
            [
              {"title": "第一财经: Q1车市价格战复盘，新能源市场热度不减", "url": "...", "snippet": "..."},
              {"title": "懂车帝: 2024年3月新能源汽车销量榜与用户关注度分析", "url": "...", "snippet": "..."},
              {"title": "微博话题#新能源车主续航焦虑#阅读量破亿", "url": "...", "snippet": "..."}
            ]
            ```
    *   **c. 生成初始总结 (`first_summary_node`)**:
        *   **输入**: 段落标题、搜索查询、以及上述搜索结果的文本片段。
        *   **LLM指令**: > "根据以下搜索结果，为'市场总体热度与关注度变化'这个主题写一段总结。"
        *   **模拟输出 (总结文本)**:
            > "2024年第一季度，新能源汽车市场保持了极高的舆论热度。各大媒体报道显示，尽管存在续航焦虑等负面讨论，但由'价格战'引发的市场竞争持续吸引着公众的广泛关注。根据懂车帝等平台的数据，3月份的用户关注度和销量均达到季度高点。"
4.  **状态更新**: `state.paragraphs[0].research.search_history` 记录了这次搜索的关键词和结果。`state.paragraphs[0].research.latest_summary` 被更新为上述总结文本。

#### **2.2 `_reflection_loop` - 反思与补充搜索**

1.  **调用线索**: `_process_paragraphs()` -> `_reflection_loop()` -> `reflection_node.run()` -> `execute_search_tool()` -> `reflection_summary_node.mutate_state()`
2.  **核心节点**: `ReflectionNode`, `ReflectionSummaryNode`
3.  **内部逻辑与模拟输出**:
    *   **a. 生成反思查询 (`reflection_node`)**:
        *   **输入**: 段落标题和**当前的总结文本**。
        *   **LLM指令**: > "这是关于'市场总体热度与关注度变化'的现有总结：'[...总结文本...]'。请审视这份总结，判断是否完整，是否缺少关键信息或数据支撑？我下一步应该搜索什么来改进它？请以JSON格式返回'reasoning'和'search_query'。"
        *   **模拟输出**:
            ```json
            {
              "reasoning": "当前总结提到了热度高点，但缺乏具体的月度对比数据来支撑'变化趋势'。我应该寻找更精确的数据，如搜索引擎指数。",
              "search_query": "新能源汽车 2024年1月 2月 3月 百度指数",
              "search_tool": "web_search_only"
            }
            ```
    *   **b. 执行反思搜索 (`execute_search_tool`)**:
        *   **输入**: `tool_name="web_search_only"`, `query="新能源汽车 2024年1月 2月 3月 百度指数"`。
        *   **模拟输出 (搜索结果)**:
            ```json
            [
              {"title": "百度指数 - '新能源汽车'关键词趋势", "url": "...", "snippet": "数据显示，1月平稳，2月春节期间下跌15%，3月因新车发布和促销活动，指数环比上涨40%。"}
            ]
            ```
    *   **c. 生成反思总结 (`reflection_summary_node`)**:
        *   **输入**: **原始总结** + **新的搜索结果**。
        *   **LLM指令**: > "请使用以下新信息：'[...百度指数的结果...]'，来更新和完善这段总结：'[...原始总结...]'。"
        *   **模拟输出 (更新后的总结文本)**:
            > "2024年第一季度，新能源汽车市场舆论热度呈现波动上升的态势。根据百度指数数据，相关关键词搜索量在2月春节期间有短暂回落，但随后的3月份因多家品牌发布新车和开启新一轮'价格战'，关注度迅速反弹并达到季度峰值，环比增长达40%。这表明市场竞争是驱动舆论热度的核心因素。"
4.  **状态更新**: `state.paragraphs[0].research.latest_summary` 被**覆盖**为这段更完善、有数据支撑的总结。这个反思循环可能会进行多次（由`MAX_REFLECTIONS`配置决定）。

---

### **第三步: `_generate_final_report` - 生成最终报告**

1.  **调用线索**: `research()` -> `_generate_final_report()` -> `report_formatting_node.run()`
2.  **核心节点**: `ReportFormattingNode`
3.  **输入**:
    *   所有段落的**最终版**总结文本。
    *   报告标题: `"2024年第一季度新能源汽车市场舆论趋势分析报告"`
4.  **内部逻辑**:
    *   `ReportFormattingNode` 调用 LLM，指令类似：
        > "请将以下各章节内容整合成一份完整的、格式优美的Markdown研究报告。报告标题为'...'。请确保章节过渡自然，并可以适当添加引言和结论。章节1标题：'...'，内容：'...'。章节2标题：'...'，内容：'...'..."
5.  **模拟输出 (最终的Markdown报告字符串)**:
    ```markdown
    # 2024年第一季度新能源汽车市场舆论趋势分析报告

    ## 引言
    本报告旨在深度分析2024年第一季度中国新能源汽车市场的公众舆论态势...

    ## 一、市场总体热度与关注度变化
    2024年第一季度，新能源汽车市场舆论热度呈现波动上升的态势...

    ## 二、主要品牌声量与口碑对比
    （这是处理第二个段落后生成的总结）...

    ...

    ## 结论
    总体来看，2024年Q1新能源市场在激烈的竞争中保持了高热度...
    ```
6.  **状态更新**: `state.final_report` 被赋值为这个完整的Markdown字符串。

---

### **第四步: `_save_report` - 保存报告**

1.  **调用线索**: `research()` -> `_save_report()`
2.  **输入**: 上一步生成的完整Markdown报告。
3.  **内部逻辑**:
    *   生成一个文件名，如 `deep_search_report_分析一下2024年第一季度_20240521_150000.md`。
    *   将Markdown字符串写入到 `OUTPUT_DIR` 目录下的这个文件中。
4.  **最终输出**: 在文件系统中创建了一个 `.md` 文件，并将报告内容返回给最初的调用者。
